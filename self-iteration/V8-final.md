# 自我迭代实验模式（V8）最终版

> 一句话定位：通过自我批评 + 结构化迭代，持续优化提示词/规则
> 适用场景：提示词工程、自动化规则设计、知识库迭代

---

## 🚀 快速开始

**Step 1：判断任务类型**
| 类型 | 模式 | 推荐轮数 |
|------|------|----------|
| 全新设计 | 标准 | 5-8 |
| 迭代优化 | 标准 | 3-5 |
| 快速修复 | 轻量 | 1-2 |
| 微调实验 | 轻量 | 1-3 |

**Step 2：选择模式**
- 标准模式：完整5维度评估（创新性/实用性/安全性/简洁性/稳定性）
- 轻量模式：满意度1-5 + "解决了什么问题？"

**Step 3：开始迭代**

---

## 核心规则

1. **单任务专注**：一次只迭代一个目标
2. **智能终止**：连续2轮无实质提升 → 停止
3. **失败识别**：连续3轮无新缺点 → 强制停止
4. **弹性加速**：第4轮起要求新角度，但允许"无新角度"声明

---

## 每轮输出模板

```
【迭代 N/X】当前版本

【核心差异】vs 初始版本的关键差异

【价值评估】
- 标准：5维度评分（每维度1-5）
- 轻量：满意度 1-5
- 必答：这个版本解决了什么问题？

【自我批评】
- 缺点≥4条（必须尖锐）
- 优点1条（核心价值）

【改进计划】具体修改策略

【案例参考】（可选）之前有类似改进吗？

【下一轮输出】完整新版本
```

---

## 5维度评分标准

| 维度 | 1分 | 3分 | 5分 |
|------|-----|-----|-----|
| 创新性 | 模仿/微调 | 有差异化的改进 | 原创方法 |
| 实用性 | 概念不可落地 | 可执行但有障碍 | 直接可用 |
| 安全性 | 有明显漏洞/风险 | 潜在风险可缓解 | 无已知风险 |
| 简洁性 | 过度复杂难以理解 | 结构清晰但可简化 | 恰到好处 |
| 稳定性 | 容易死循环/崩溃 | 有保护但不够完善 | 健壮 |

```
--- 🛑 人类反馈点 ---
进度：N/X

上轮建议吸收情况：Y/N + 理由

```

---

## 终止与总结

触发终止时输出：
1. **最终版本**：完整可用的版本
2. **进化树**：各维度的最佳版本
3. **关键突破**：相比初始版本的核心提升
4. **经验总结**：本轮迭代学到了什么

## 执行约束

- 用当前模型运行
- 3轮后统一git commit（repo: openclaw-self-iteration-experiments）
- 允许修改本规则，说明理由即可
- 语法检查：新版本必须可被正确解析

## V0 → V8 迭代历程

- V0：初始版本
- V1：灵活轮数、终止条件、容错、迭代历史
- V2：5维度评分、自检确认、最佳候选
- V3：场景化权重、评分锚点、人类反馈哨兵、轻量模式
- V4：快速入门、版本对比表、改进日志、冒烟测试
- V5：精简输出区块、失败识别、外部参考
- V6：轻量评分卡、脉络回顾、迭代加速器、案例沉淀
- V7：任务类型预判、弹性加速器、案例调用、语法检查
- V8：最终完善版

---

## 关键提升总结

| 维度 | V0 | V8 |
|------|-----|-----|
| 迭代灵活性 | 固定5轮 | 任务类型预判 + 弹性轮数 |
| 终止机制 | 无 | 智能终止 + 失败识别 |
| 质量评估 | 无 | 5维度评分（标准）/满意度（轻量） |
| 迭代效率 | 无 | 迭代加速器 + 案例库调用 |
| 人类交互 | 无 | 人类反馈哨兵 + 建议历史 |
| 防错机制 | 无 | 语法检查 + 弹性加速 |
| 可用性 | 单一模板 | 快速开始 + 任务匹配 |

**关键技巧：**
1. 任务预判 → 让迭代长度匹配任务复杂度
2. 评分锚点 → 把主观评价变得可量化
3. 失败识别 → 防止无效循环
4. 案例复用 → 不重复造轮子
5. 弹性约束 → 既要多样性，又不强求
